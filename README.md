##  Tema & Cerita Studi Kasus

### Tema Kasus: 
**Digitalisasi Manajemen Kedisiplinan Siswa Berbasis Poin (Point-Based Disciplinary System)**

### Cerita Studi Kasus:
**SMA Negeri 1 Prestasi** adalah salah satu sekolah unggulan yang memiliki standar kedisiplinan yang tinggi. Namun, selama bertahun-tahun, pengelolaan data pelanggaran siswa masih dilakukan secara konvensional menggunakan buku besar dan kartu kendali fisik.

**Masalah Utama:**
1. **Redundansi Data:** Seringkali terjadi kesalahan pencatatan poin yang sama di buku yang berbeda.
2. **Keterlambatan Informasi:** Orang tua dan wali kelas sulit mendapatkan data akumulasi poin secara cepat.
3. **Pemberian Sanksi Tidak Akurat:** Akumulasi poin seringkali tidak terhitung dengan tepat, sehingga pemberian Surat Peringatan (SP) sering terlambat diberikan.
4. **Keamanan Data:** Buku fisik rentan rusak atau hilang.

**Solusi Sistem (E-BK):**
E-BK (Sistem Informasi Disiplin Siswa) dibangun untuk mentransformasi proses manual tersebut menjadi digital. Sistem ini memungkinkan Guru BK atau Admin untuk:
- Mencatat pelanggaran secara instan.
- Menghitung poin secara otomatis berdasarkan jenis pelanggaran.
- Memberikan peringatan otomatis saat siswa mencapai ambang batas poin tertentu (SP1, SP2, SP3).
- Mencetak laporan dan surat teguran secara profesional dalam hitungan detik.

Dengan E-BK, sekolah dapat mewujudkan ekosistem pendidikan yang lebih transparan, akuntabel, dan efisien dalam membentuk karakter siswa yang disiplin.

---

##  Fitur Utama
- **Dashboard Informatif:** Statistik total siswa, pelanggaran, dan rangking poin tertinggi.
- **Manajemen Data:** CRUD Data Siswa, Guru, Kelas, dan Angkatan.
- **Input Pelanggaran:** Form input pelanggaran yang terintegrasi dengan poin otomatis.
- **Log Aktivitas:** Riwayat pelanggaran siswa yang terdokumentasi dengan baik.
- **Cetak Surat:** Fitur cetak surat peringatan (SP) langsung dari sistem.
- **Role Access:** Perbedaan hak akses antara Admin TU dan Guru BK.

---

## Teknologi yang Digunakan
- **Core Framework:** CodeIgniter 3 (MVC Architecture)
- **Frontend:** Bootstrap 5, FontAwesome, JQuery
- **Database:** MySQL / MariaDB
- **UI/UX:** SweetAlert2 (Notifikasi), DataTables (Manajemen Tabel)

---

## Instalasi
1. Clone repositori ini.
2. Import database `e-bk.sql` ke MySQL.
3. Sesuaikan konfigurasi database di `application/config/database.php`.
4. Jalankan pada server lokal (XAMPP/Laragon).
